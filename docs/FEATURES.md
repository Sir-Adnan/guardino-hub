# Guardino Hub Features

## نقش‌ها

سوپرادمین مدیریت نودها، رسیلرها، قیمت‌ها، موجودی، سیاست‌ها، تخصیص‌ها، گزارش‌ها و API tokenها را در اختیار دارد.

رسیلر کاربران، عملیات سرویس، سفارش‌ها، کیف پول و لینک‌های مربوط به محدوده خودش را مدیریت می‌کند.

## پنل‌های پشتیبانی‌شده

| Panel | قابلیت‌های فعال |
| --- | --- |
| PasarGuard | ساخت و مدیریت کاربر، sync مصرف و وضعیت، import کامل، لینک اشتراک و credential اختصاصی |
| Marzban | ساخت و مدیریت کاربر، sync مصرف و وضعیت، import کامل، لینک اشتراک و credential اختصاصی |
| WGDashboard | مدیریت WireGuard، config، مصرف، revoke و reset |

## نود و تخصیص

هر تخصیص یک رسیلر را به یک نود متصل می‌کند و شامل وضعیت فعال، نود پیش‌فرض، قیمت override و حالت credential است:

- `shared`: استفاده از credential اصلی نود
- `dedicated`: استفاده از username/password یا token همان رسیلر

credential اصلی PasarGuard می‌تواند حساب sudo باشد و credential اختصاصی رسیلر می‌تواند حساب operator محدود به کاربران همان رسیلر باشد.

برداشتن تخصیص یا حذف نود، نگاشت‌های محلی را از دید پنل خارج می‌کند و عملیات حذف remote را اجرا نمی‌کند.

## ورود کاربران موجود

- دریافت تمام صفحات کاربران PasarGuard و Marzban
- ورود اولیه بدون سفارش، ledger یا کسر موجودی
- تشخیص کاربران موجود و جلوگیری از duplicate
- بازیابی user قدیمی Guardino و حفظ `id`، زمان ساخت و سابقه سفارش
- ادغام duplicate خارجی با رکورد مالی اصلی در حالت قابل تشخیص
- محدودسازی جزئیات response برای جلوگیری از payload بسیار بزرگ

## Sync

- پردازش batch برای پنل‌های پرتعداد
- گروه‌بندی درخواست‌ها بر اساس credential نود یا تخصیص
- توقف امن در timeout و پاسخ ناقص
- تشخیص bulk list غیرقابل اعتماد
- تأیید مستقیم چندمرحله‌ای پیش از حذف remote
- نگهداری آخرین مصرف معتبر هنگام اختلال پنل
- snapshot روزانه برای داشبورد و گزارش‌ها

## عملیات کاربران

- ساخت تکی و گروهی
- انتخاب نود دستی یا پیش‌فرض
- حالت فعال و On Hold
- تمدید و افزایش حجم یا زمان
- تغییر وضعیت، reset usage و revoke
- حذف و ریفاند بر اساس سابقه مالی
- لینک ساب مرکزی و لینک مستقیم هر نود
- QR Code و کپی سریع لینک‌ها
- فیلتر وضعیت، انقضا، اتمام حجم و انتظار اتصال

## مالی و گزارش

- کیف پول رسیلر
- دفترکل تراکنش‌ها
- سفارش‌های `pending`، `completed`، `failed` و `rolled_back`
- قیمت پایه، bundle، روزانه و override هر تخصیص
- آمار کل مستقل از صفحه‌بندی جدول‌ها
- نمودار مصرف و حجم ثبت‌شده بر اساس حساب و بازه زمانی

## امنیت

- access token ورود
- API token قابل revoke
- محدودیت دسترسی بر اساس حساب
- TOTP دومرحله‌ای
- secret رمزنگاری‌شده و recovery codeهای hash‌شده
- challenge مجزا برای مرحله دوم ورود

## رابط کاربری

- زبان فارسی و انگلیسی
- انتخاب ارقام فارسی یا انگلیسی
- حالت روشن و تاریک
- صفحات واکنش‌گرا برای موبایل و دسکتاپ
- تنظیمات تب‌بندی‌شده برای ظاهر، امنیت و رفتار کاربران

## API

OpenAPI، API token و `client_request_id` اتصال ابزارهای بیرونی را پوشش می‌دهند. محدوده داده و عملیات هر token از حساب صادرکننده آن پیروی می‌کند.
