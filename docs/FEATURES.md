# Guardino Hub Features

## نقش‌ها

سوپرادمین مالک اصلی پنل است و به مدیریت نودها، رسیلرها، قیمت‌ها، موجودی‌ها، سیاست‌ها، تخصیص‌ها، گزارش‌ها و API tokenها دسترسی دارد.

رسیلر فروشنده‌ای است که با موجودی کیف پول خود کاربر می‌سازد، تمدید انجام می‌دهد، حجم/زمان اضافه می‌کند و لینک سرویس را تحویل می‌دهد.

## پنل‌های پشتیبانی‌شده

| Panel | وضعیت |
| --- | --- |
| PasarGuard | ساخت و مدیریت کاربر، sync مصرف، import کاربران، credential اختصاصی |
| Marzban | ساخت و مدیریت کاربر، لینک ساب، sync مصرف، import کاربران، credential اختصاصی |
| WGDashboard | مدیریت WireGuard، لینک config، sync مصرف و revoke |

## تخصیص نود و credential اختصاصی

هر رسیلر می‌تواند روی هر نود یک تخصیص داشته باشد. تخصیص شامل فعال/غیرفعال بودن، نود پیش‌فرض، قیمت اختصاصی و حالت credential است.

حالت‌های credential:

- `shared`: عملیات با credential اصلی نود انجام می‌شود.
- `dedicated`: عملیات همان رسیلر با username/password یا token اختصاصی خودش روی همان نود انجام می‌شود.

این مدل برای PasarGuard بسیار مهم است: credential اصلی نود بهتر است دسترسی sudo داشته باشد تا سوپرادمین بتواند کل پنل را مدیریت کند. credential اختصاصی رسیلر معمولا operator/admin معمولی است تا فقط کاربران خودش را ببیند و تغییر دهد.

## import کاربران قدیمی

وقتی برای یک رسیلر credential اختصاصی ثبت می‌شود، می‌توان کاربران قبلی همان ادمین را از PasarGuard یا Marzban وارد Guardino کرد.

قانون مالی import:

- import کاربر قدیمی سفارش و ledger جدید نمی‌سازد.
- از کیف پول رسیلر چیزی کم نمی‌شود.
- از لحظه ورود به Guardino، عملیات بعدی مثل تمدید، افزایش حجم یا تغییر پلن طبق قیمت‌ها و سیاست‌های گاردینو محاسبه می‌شود.

## عملیات کاربران

- ساخت کاربر تکی یا گروهی
- انتخاب حجم، زمان، نودهای دستی یا نودهای پیش‌فرض
- تمدید با سیاست‌های قابل تنظیم
- افزایش حجم و زمان
- حذف و ریفاند طبق سیاست
- reset usage و revoke لینک
- نمایش لینک مستقیم پنل‌ها و لینک ساب مرکزی Guardino
- نمایش مصرف‌های کوچک در حد MB
- sync وضعیت on-hold، مصرف، انقضا و حذف شدن کاربر از پنل بالادستی

## مالی و گزارش

- کیف پول رسیلر
- دفترکل تراکنش‌ها
- سفارش‌ها با وضعیت completed، pending، failed و rolled_back
- قیمت پایه، قیمت bundle، قیمت روزانه و قیمت override برای هر نود
- داشبورد فروش، حجم زده‌شده، مصرف ثبت‌شده، وضعیت کاربران و سلامت نودها
- snapshot روزانه برای مصرف و ظرفیت کلی هر رسیلر

## API و توسعه فروش

گاردینو با API token و `client_request_id` برای اتصال ابزارهای بیرونی و ربات تلگرام آماده شده است. هر ربات می‌تواند با token سوپرادمین یا token رسیلر خاص کار کند و فقط به محدوده همان حساب دسترسی داشته باشد.
