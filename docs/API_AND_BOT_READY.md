# API Integration

Guardino Hub API برای پنل، ابزارهای مدیریتی و سرویس‌های فروش بیرونی قابل استفاده است. خود پروژه ربات تلگرام اجرا نمی‌کند.

## دسترسی

- token سوپرادمین به محدوده مدیریت کل سیستم دسترسی دارد.
- token رسیلر فقط داده‌ها و عملیات همان رسیلر را می‌بیند.
- API token نمی‌تواند API token دیگری ایجاد کند.
- tokenها از پنل قابل ایجاد و revoke هستند.

OpenAPI هر نصب:

```text
https://YOUR_DOMAIN/api/openapi.json
https://YOUR_DOMAIN/api/docs
```

## قرارداد عملیات فروش

جریان API شامل دریافت اطلاعات مجاز حساب، ثبت عملیات کاربر، اجرای عملیات روی پنل بالادستی، ثبت سفارش و دفترکل و بازگرداندن نتیجه است.

عملیات مالی از `client_request_id` استفاده می‌کنند. ارسال دوباره همان شناسه، عملیات تکراری جدید ایجاد نمی‌کند و نتیجه قبلی یا وضعیت conflict را برمی‌گرداند.

## محدوده رسیلر

رسیلر می‌تواند روی نود مشترک از credential اصلی نود یا credential اختصاصی تخصیص خودش استفاده کند. در حالت dedicated:

- عملیات کاربر با credential همان تخصیص اجرا می‌شود.
- کاربران موجود همان credential به‌صورت صفحه‌بندی‌شده قابل ورود هستند.
- ورود اولیه کاربر خارجی تراکنش مالی ایجاد نمی‌کند.
- عملیات بعدی از مدل مالی Guardino عبور می‌کند.

## کاربران واردشده

ورود کاربران بر اساس remote identifier و مالک رسیلر تطبیق داده می‌شود. رکوردهای قدیمی که به‌دلیل حذف اشتباه remote مخفی شده‌اند بازیابی می‌شوند. اگر یک duplicate خارجی روی همان remote identifier وجود داشته باشد و رکورد مالی اصلی قابل تشخیص باشد، نگاشت remote به رکورد اصلی بازمی‌گردد.

## قرارداد مالی

- ساخت، تمدید و افزایش حجم سفارش و ledger ثبت می‌کنند.
- ورود اولیه کاربر خارجی سفارش مالی ایجاد نمی‌کند.
- خطای عملیات remote پیش از ثبت نهایی مالی به وضعیت failed یا rollback منتهی می‌شود.
- ریفاند کاربر ساخته‌شده در Guardino از سابقه سفارش create و قیمت snapshot استفاده می‌کند.
- دسترسی مستقیم به ledger از API عمومی ارائه نمی‌شود.

## امنیت

- محدوده token در backend کنترل می‌شود.
- 2FA ورود تعاملی را پوشش می‌دهد؛ API token مستقل و قابل revoke است.
- اطلاعات credential تخصیص فقط در مسیرهای مدیریتی استفاده می‌شود.
- token و credential در پاسخ‌های عمومی یا لاگ‌های عادی نمایش داده نمی‌شوند.

## داده‌های قابل استفاده

API فعلی عملیات کاربران، لینک‌ها، نودهای مجاز، تخصیص‌ها، کیف پول، سفارش‌ها، دفترکل، آمار داشبورد و تنظیمات حساب را پوشش می‌دهد. نام دقیق routeها و schemaها از OpenAPI همان نسخه نصب‌شده خوانده می‌شود.
